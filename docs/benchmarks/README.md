# Benchmark report

## Goal

Compare runtime behavior across PyTorch, ONNX Runtime, CPU/GPU, and API-oriented settings.

## Dataset

- Default benchmark input source: processed BraTS cases under data/processed/Task01_BrainTumour.
- Labels are optional and used when available to compute foreground Dice.
- Benchmark output is intended for runtime comparison, not for clinical quality claims.

## Model

- Reference checkpoint is passed via --checkpoint.
- ONNX benchmark can use --onnx_model or auto-export from the checkpoint.

## Run

```bash
python src/benchmark.py \
  --checkpoint outputs/Task01_2d_axi_jun03/checkpoints/ckpt_best.pt \
  --input_dir data/processed/Task01_BrainTumour \
  --labels_dir data/processed/Task01_BrainTumour \
  --backends pytorch_cpu onnx_cpu \
  --output docs/benchmarks/results.csv
```

## Results

Artifacts generated by the benchmark command:

- results.csv
- results.md
- environment.json

## Interpretation

Use the table to compare latency, memory, and model-size tradeoffs across backends.

## Limitations

- Small benchmark size can bias latency estimates.
- Not clinically validated.
- Runtime depends on preprocessing and hardware.
- Quantization may reduce segmentation quality.
